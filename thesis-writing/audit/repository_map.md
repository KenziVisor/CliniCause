## Important Structural Facts

- `final-results/` is the largest result archive but is ignored and untracked. A thesis should cite it only with an added manifest, commit pointer, checksum list, or equivalent archival decision.
- The causal final archive contains exactly twelve parseable run folders: dataset in `{mimic, physionet}`, estimator in `{forest, linear, pfn}`, and sampling in `{no-downsample, downsample}`.
- `run_summary.json` files use absolute `/truenas/home/kenzikob/...` paths for source repo, output root, config CSVs, latent-tags directories, and processed pickles. STraTS summaries/logs use absolute `/workspace/...` output paths. These paths reduce portability.
- Numbered config CSVs referenced by the final causal runs, such as `configs/mimic-global-variables-5.csv` and `configs/physionet-global-variables-6.csv`, are not present in the repository or `final-results/`.
- Multiple versions of similar workflows coexist: parent `router.py`, causal `main.py`, shell wrappers, STraTS shell wrappers, active taggers, older taggers, and `src/draft/`.
- Matching outputs are repeated under forest, linear, and PFN run folders for the same dataset/sampling, even though matching does not depend on the CATE estimator. The cross-run script deduplicates them.
- PFN runs have successful graph, majority vote, mortality prediction, matching, and CATE stages, but sensitivity analysis and permutation tests are intentionally skipped by `run_summary.json`.
- Decision-tree pickles, model pickles, `checkpoint_best.bin`, and `pt_saved_variables.pkl` are binary support artifacts. They were inventoried but not deserialized.
- Raw data and processed dataset pickles required for reproduction are not locally available as tracked repository files.
- Prompt artifacts are design-provenance evidence only. Active tagger and graph source files define implemented proxy states and DAGs; prompt outputs do not prove clinical validation or causal discovery.
