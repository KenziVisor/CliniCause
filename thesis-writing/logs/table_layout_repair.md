## Defects found and repairs applied

| Tables | Defect found | Repair |
| --- | --- | --- |
| Front-matter longtables | Justified paragraph cells produced uneven word spacing and underfull-box diagnostics. | Switched the two column definitions to the local ragged-right `L{}` paragraph-column type. |
| 1.1, 3.1, 4.1 | Uneven justification in narrow paragraph columns. | Switched affected `p{}` columns to `L{}`. |
| 2.1--2.2 | Portrait layout forced narrow, highly expanded cells and poor header/value readability. | Used a local landscape page, `\small`, narrower inter-column spacing, and redistributed ragged-right widths. |
| 5.1, 5.3 | Dense columns and table-width pressure. | Used ragged-right paragraph columns and local `\tabcolsep` reduction. |
| 5.2 | Text overlapped across cells around long artifact identifiers; headers and content were too dense in portrait. | Used local landscape layout, redistributed widths, and inserted safe break points in the two long PDF identifiers. |
| 5.4 | Unreadably compressed portrait columns and a long construct name that crossed its available width. | Used local landscape layout, redistributed widths, and inserted a safe break after `Hematologic/`. |
| 5.5 | Unreadably compressed portrait columns. | Used local landscape layout and redistributed widths. |
| 6.1--6.2 | Uneven justification in paragraph columns. | Switched affected columns to `L{}`. |
| 7.1--7.4 | Uneven justification; `OUT_InHospitalMortality` and `CausalForestDML` produced table-local overfull boxes. | Used `L{}` columns; made the identifiers safely breakable without changing their content. |
| 8.1--8.3 | Dense multi-column portrait layouts with poor wrapping. | Used local landscape layout, `\small`, compact column spacing, and redistributed `L{}` widths. |
| 9.1 | Dense multi-column portrait layout. | Used local landscape layout, compact spacing, and redistributed `L{}` widths. |
| 9.2 | Eight-column longtable exceeded its line width and was difficult to read. | Used local landscape layout, `\footnotesize`, compact spacing, and widths sized to the available longtable line. |
| 9.3--9.4 | Uneven justification in paragraph columns. | Switched affected columns to `L{}`. |
| 10.1--10.2 | No visible defect. | Inspected; no change required. |
| 10.3--10.8 | Uneven justification in paragraph columns. | Switched affected columns to `L{}`. |
| 11.1--11.2 | Uneven justification in paragraph columns. | Switched affected columns to `L{}`. |

The shared `L{}` type is a ragged-right `p{}`-style column.  `pdflscape` was added only to support the tables listed above that genuinely require landscape width.

## Validation

- Build command: `latexmk -xelatex main.tex` from `thesis-writing/thesis`.
- Build result: successful; generated a 133-page PDF.
- Log review: no table-local overfull or underfull box diagnostics remain.  Remaining overfull-box messages are in pre-existing prose, headings, or bibliography content rather than table layout.  `xdvipdfmx` duplicate-destination warnings are not layout warnings and did not affect rendered tables.
- Visual inspection: all 30 labeled tables and both front-matter longtables were rendered at 300 DPI and inspected.  No overlap, cell-boundary crossing, clipping, caption collision, page-margin overflow, or unreadable column width remains.
- Content preservation: source diff review confirms the changes are table formatting, local orientation, spacing, and safe discretionary breaks only; values and table content are unchanged.
