# Stage 4.0 — Prepare the Thesis-Writing Environment and LaTeX Skeleton

## Context

Stages 0–3 of the CliniCause thesis workflow are complete.

The repository should already contain:

* the validated literature corpus under `thesis-writing/literature/`;
* the repository and evidence audit under `thesis-writing/audit/`;
* the approved thesis outline and construction plan under `thesis-writing/planning/`;
* the BGU thesis instructions and example thesis/template resources;
* the research repositories and available result artifacts.

This task begins Stage 4, but it must **not write substantive thesis prose**. Its purpose is to convert the approved Stage 3 plan into a safe, organized, compilable LaTeX workspace.

## Objective

Create or adapt the thesis LaTeX structure so that subsequent Stage 4 prompts can draft individual chapters without reorganizing the project, breaking compilation, duplicating bibliography files, or inventing content.

The output should be a clean thesis skeleton with:

* approved chapter and section files;
* front matter and back matter;
* bibliography integration;
* stable labels and file naming;
* placeholder conventions;
* compilation instructions;
* an initial clean compile, or a precise report of compilation blockers.

## Required Initial Inspection

Before editing anything:

1. Confirm the repository root.
2. Run and record:

```bash
git status --short
git branch --show-current
git log -1 --oneline
```

3. Inspect all files under:

```text
thesis-writing/planning/
thesis-writing/audit/
thesis-writing/literature/
```

4. Locate and inspect:

* the approved thesis outline;
* chapter evidence map;
* writing order;
* terminology and notation conventions;
* figure and table plans;
* citation plan;
* the BGU thesis instructions;
* the existing thesis LaTeX source, if present;
* the example thesis/template files;
* existing bibliography configuration;
* any existing build scripts, `latexmkrc`, Makefiles, or compilation instructions.

5. Determine whether the thesis workspace already exists and whether it compiles before modification.

Do not assume that filenames or compile commands are correct merely because they exist.

## Source-of-Truth Priority

When sources disagree, use this priority:

1. approved Stage 3 planning documents;
2. BGU thesis instructions;
3. current thesis LaTeX files;
4. repository audit outputs;
5. literature corpus metadata;
6. example thesis materials.

The example thesis is a formatting reference only. Its prose, claims, results, figures, tables, personal details, and bibliography must not be copied into the CliniCause thesis.

## Allowed Scope

You may inspect:

```text
the full repository
thesis-writing/planning/
thesis-writing/audit/
thesis-writing/literature/
thesis-writing/general-instructions.pdf
thesis-writing/example-omri-thesis/
existing thesis/template/build files
```

You may edit or create files only inside:

```text
thesis-writing/thesis/
thesis-writing/logs/
```

A root-level or thesis-local build configuration may be edited only when necessary for thesis compilation and only if it is clearly part of the thesis workspace.

## Forbidden Scope

Do not:

* write substantive chapter prose;
* summarize research findings in paragraph form;
* invent results, methods, citations, figures, tables, or contributions;
* edit research code;
* edit datasets or result artifacts;
* rerun preprocessing, model training, or causal experiments;
* modify the approved audit, planning, or literature files;
* rename existing bibliography keys;
* create a second independent bibliography database;
* copy prose or research content from the example thesis;
* silently delete existing thesis material;
* commit, push, merge, rebase, or modify Git history.

## Required Procedure

### 1. Determine the Existing Thesis State

Classify the current thesis workspace as one of:

* usable existing thesis;
* partially usable thesis;
* template only;
* no thesis workspace.

Record the classification and supporting evidence in the setup report.

When existing thesis content is found:

* preserve it;
* do not overwrite it blindly;
* report conflicts with the approved Stage 3 outline;
* move or rename files only when clearly necessary;
* document every structural change.

### 2. Establish the Thesis Directory Structure

Use the existing structure when it is sound. Otherwise, create a clear structure similar to:

```text
thesis-writing/thesis/
├── main.tex
├── frontmatter/
│   ├── title_pages.tex
│   ├── abstract_primary.tex
│   ├── abstract_secondary.tex
│   ├── keywords.tex
│   ├── acknowledgements.tex
│   └── nomenclature.tex
├── chapters/
│   ├── 01_introduction.tex
│   ├── 02_background_related_work.tex
│   ├── 03_problem_formulation.tex
│   ├── 04_data_preprocessing.tex
│   ├── 05_predictive_modeling.tex
│   ├── 06_proxy_state_construction.tex
│   ├── 07_causal_methodology.tex
│   ├── 08_experimental_design.tex
│   ├── 09_results.tex
│   ├── 10_discussion.tex
│   └── 11_conclusion.tex
├── appendices/
│   └── appendices.tex
├── figures/
├── tables/
├── references.bib
├── latexmkrc
└── README.md
```

This is a suggested structure, not a command to replace a valid existing template. Follow the approved Stage 3 outline and the template’s architecture.

Do not create empty chapter files that conflict with an existing include system.

### 3. Create the Approved Outline in LaTeX

Translate the approved Stage 3 outline into:

* `\chapter`;
* `\section`;
* `\subsection`;

headings only.

Do not create speculative subheadings that were not approved.

Each planned section should contain only a visible placeholder comment, for example:

```latex
% [STAGE 4 DRAFT REQUIRED]
% Evidence map: see thesis-writing/planning/chapter_evidence_map.md
```

Do not insert filler prose such as “This chapter discusses...”.

### 4. Establish Stable Labels

Create a consistent label convention, such as:

```latex
chap:
sec:
subsec:
fig:
tab:
eq:
app:
```

Add labels to approved chapters and sections where appropriate.

Do not rename valid existing labels unless they are duplicated, malformed, or inconsistent with the approved structure. Record every renamed label and update all internal references.

### 5. Integrate the Canonical Bibliography

The canonical bibliography source is:

```text
thesis-writing/literature/metadata/references.bib
```

Use one of these approaches:

* reference it directly from the thesis build; or
* create a documented symbolic link or synchronized thesis-local reference, if required by the toolchain.

Do not manually create a divergent duplicate.

Verify:

* the selected bibliography backend;
* the bibliography style;
* compatibility with the template;
* that existing citation keys remain unchanged;
* that a minimal bibliography build succeeds.

Do not add uncited placeholder citations merely to force entries into the bibliography.

### 6. Configure Front Matter

Prepare structural files for the required front matter, based on the approved thesis language and BGU instructions.

The structure must support:

* title and approval pages;
* primary-language abstract;
* secondary-language abstract;
* abstracts limited to 500 words when later drafted;
* 10–15 keywords or key phrases;
* acknowledgements;
* table of contents through the third heading level;
* nomenclature or list of symbols, when required;
* list of figures;
* list of tables;
* correct front-matter and main-matter page-numbering transitions.

Use placeholders for personal, administrative, or approval information that has not been supplied:

```text
[AUTHOR DETAILS REQUIRED]
[SUPERVISOR DETAILS REQUIRED]
[DEPARTMENT DETAILS REQUIRED]
[APPROVAL TEXT REQUIRED]
```

Do not infer these values.

### 7. Configure Figures and Tables

Create stable thesis-local figure and table paths without copying unrelated example assets.

Where the Stage 3 plans identify approved source assets:

* add comments documenting their expected source paths;
* do not copy or regenerate the assets unless required for a compile test;
* do not include figures or tables in chapters yet unless they are already approved and part of the existing skeleton.

Create placeholder conventions for later prompts, for example:

```latex
% [FIGURE REQUIRED: figure_plan identifier]
% [TABLE REQUIRED: table_plan identifier]
```

### 8. Establish Placeholder Conventions

Use only the following visible placeholder classes unless the planning documents define stricter ones:

```text
[STAGE 4 DRAFT REQUIRED]
[RESULT REQUIRED]
[CITATION REQUIRED]
[FIGURE REQUIRED]
[TABLE REQUIRED]
[VALIDATION REQUIRED]
[SUPERVISOR DECISION REQUIRED]
[ADMINISTRATIVE DETAILS REQUIRED]
```

Create or update:

```text
thesis-writing/logs/unresolved_placeholders.md
```

The file should explain each placeholder class and initially inventory any placeholders already present in the thesis.

Do not hide unresolved material in generic TODO comments.

### 9. Create Thesis Build Instructions

Create or update:

```text
thesis-writing/thesis/README.md
```

Document:

* required LaTeX engine;
* bibliography backend;
* exact clean-build command;
* incremental-build command;
* cleanup command;
* expected output PDF path;
* required packages or system tools;
* known warnings that are currently acceptable;
* working-directory assumptions.

Prefer an automated command such as `latexmk` when compatible with the template.

### 10. Compile and Validate

First run a non-destructive baseline build when an existing thesis is present.

After structural changes:

1. clean generated auxiliary files;
2. run the documented full compile;
3. confirm that bibliography processing executes;
4. confirm that the output PDF is created;
5. inspect the compile log for:

   * undefined control sequences;
   * missing files;
   * missing citations;
   * undefined references;
   * duplicate labels;
   * bibliography errors;
   * broken figure paths;
   * fatal package conflicts.

Do not spend this task polishing harmless typography warnings such as minor overfull boxes unless they prevent meaningful validation.

When compilation cannot succeed because of an external dependency, record:

* the exact command;
* the exact error;
* the missing dependency or file;
* the narrowest proposed remedy.

Do not claim a successful compile unless the expected PDF was actually generated.

### 11. Record Deferred Fixes

Create or update:

```text
thesis-writing/logs/deferred_fixes.md
```

Record repository, template, tooling, bibliography, or structural issues discovered during this task that are real but not necessary to fix during Stage 4.0.

For each item include:

```text
ID
area
path
issue
evidence
impact
recommended later action
urgency
```

Do not fix unrelated repository issues during this task.
