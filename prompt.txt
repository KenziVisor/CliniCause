# 18. C7.2 — Adjustment-Set Logic

## 18.1 Exposure and Outcome

Define:

[

## 18.2 Available Variable Mapping

* only graph nodes with corresponding available dataframe variables can enter the observed adjustment matrix;
* missing graph candidates are recorded rather than silently created;
* graph treatment and outcome nodes are excluded from candidate adjustment sets.

## 18.3 Allowed Adjustment Universe

Verify and explain that the implementation permits only available nodes classified as:

```text
background
latent/proxy
```

Observed measurements, care-process nodes, missingness nodes, and outcome nodes are not automatically included in the adjustment pool.

Explain the rationale and limitation:

* restricting observed manifestations may reduce adjustment for descendants or mediators;
* it can also omit genuine baseline confounders not represented as allowed graph nodes;
* the safety of a proxy-state covariate depends on the exposure-specific graph.

## 18.4 Candidate Backdoor Pool

Translate the implementation accurately.

The candidate pool is formed from available permitted graph nodes that:

* are ancestors of the exposure in the original graph;
* remain ancestors of the outcome in the graph with treatment-outgoing edges removed;
* are not descendants of the exposure;
* are not the exposure or outcome.

Do not call the treatment-outgoing-edge-removed graph the ordinary `do(A)` graph.

Use notation such as:

[
\mathcal C(A,Y)
===============

\mathcal A_{\mathrm{allowed}}
\cap \operatorname{An}*{G}(A)
\cap \operatorname{An}*{G_{\underline A}}(Y)
\setminus \operatorname{De}_{G}(A),
]

only when the notation accurately reflects the source.

Define (G_{\underline A}) explicitly as the graph obtained by deleting arrows emanating from (A).

## 18.5 Backdoor Paths and Path Activity

Explain that the code:

* enumerates simple undirected paths between exposure and outcome;
* retains paths whose first edge points into the exposure;
* evaluates path activity using collider and non-collider rules;
* checks whether a candidate set blocks all encoded backdoor paths.

Avoid claiming that exhaustive enumeration is computationally scalable to arbitrary large DAGs. It is used on the project’s compact graphs.

## 18.6 Expanded-Safe Set

Explain the verified expanded mode:

1. begin from the candidate pool;
2. identify collider nodes on encoded backdoor paths;
3. remove those colliders;
4. remove their descendants;
5. test whether the remaining set blocks all encoded backdoor paths;
6. fall back to the minimal-set routine when needed.

Do not imply that simply removing every collider descendant is a universal causal-identification algorithm outside the encoded project graph.

## 18.7 Minimal Set

Explain that the minimal routine:

* starts with the full candidate pool;
* checks whether it blocks all encoded paths;
* returns failure when the full pool cannot do so;
* removes redundant variables one by one while blocking remains;
* produces a deterministic inclusion-minimal set under its ordering.

Use “inclusion-minimal under the implemented elimination order,” not “the unique minimal adjustment set.”

## 18.8 Observed Adjustment Columns
