# 1. Objective

Construct a traceable, machine-readable and human-readable results package that:

1. inventories every thesis-relevant result artifact;
2. computes stable hashes and structural metadata;
3. identifies the producing run and experiment family;
4. validates path existence and artifact schemas;
5. identifies canonical source artifacts when duplicates exist;
6. distinguishes primary/full summaries from reduced manager summaries;
7. validates predictive metrics against archived summaries and logs;
8. validates proxy-result artifacts without recomputing missing results;
9. validates causal cross-run tables against their per-run source artifacts;
10. distinguishes original and outcome-downsampled populations;
11. distinguishes estimator-native, saved-training, recomputed, fallback, skipped, and failed diagnostics;
12. prepares checked numerical CSVs for later thesis tables;
13. records unresolved scientific and advisor decisions separately from artifact validity;
14. produces a nonbinding recommended results hierarchy;
15. prevents Chapter 10 from using unverified, duplicated, mislabeled, or provenance-incomplete values without explicit qualification.

No research experiment may be rerun.

No missing result may be regenerated.

No thesis result prose may be drafted.

## 6.3 Hashing

Compute SHA-256 for:

* every manifest-listed source artifact;
* every generated checked CSV;
* each human-readable Stage 4.6A result file.

```text
<sha256>  <relative-path>
```

Do not hash temporary build files.

## 6.4 Structural Metadata

For CSV and TSV files, record:

* row count excluding header;
* column count;
* exact ordered headers;
* duplicate-header status;
* duplicate-row status when practical;
* parsing errors;
* empty-file status.

For JSON files, record:

* top-level type;
* principal key names;
* parse status;
* run/stage status fields.

For text summaries, record:

* parse status;
* expected field presence;
* relevant source or command lines;
* whether a paired log exists.

For PNG files, record:

* width;
* height;
* file mode when available;
* source relation;
* whether it is a direct archived figure or a future generated figure.

For binary artifacts, record only:

* path;
* size;
* hash;
* expected role;
* trusted reader/writer source when known.

Do not deserialize:

```text
*.pkl
*.pickle
*.bin
*.pt
*.pth
```

unless an essential value is available nowhere else and the file is a trusted project artifact. Any deserialization must be explicitly documented. Prefer not to deserialize any binary artifact in this stage.
