
must be supported.

However, for this preset only, the `thesis-main` invocation must run only these causal stages:

```text

```

It must then stop successfully.

The canonical final causal-dataset artifact must be:


For `--dataset both`, each dataset child must independently produce its own majority-vote CSV under its own isolated run directory. Preserve the existing two-process coordinator architecture and all dataset isolation guarantees.

`STAGES=all` must remain exactly equivalent to the current complete pipeline and must continue running every causal stage.

Explicit existing stage selectors must retain their current behavior unless a change is strictly required to implement the preset.

Treat `dataset-extraction` as a named preset or execution mode, not as a ninth physical stage appended to `STAGE_ORDER`.

Introduce an explicit representation that preserves both:

1. the expanded top-level router stages;

For example, after parsing, the router may retain information equivalent to:

```python
args.stage_mode = "all" | "dataset-extraction" | "explicit"
args.stages = [...]
```

The exact internal design may differ, but do not infer dataset-extraction mode later merely by comparing stage lists. The distinction must survive:

* argument parsing;
* plan fingerprinting;
* manifests;
* coordinator-to-dataset-child invocation;
* dry-run;
* validate-only;
* normal single-dataset execution;
* `--dataset both`.

Required preset expansion:

```python
dataset-extraction -> list(STAGE_ORDER)
```

The top-level sequence includes `thesis-main`; the difference is the causal stopping point passed to causal `main.py`.

Update supported-selector validation so:

```text

```

is accepted.

Reject combinations such as:

```text

```

with a clear validation error.

Preserve current comma-separated explicit-stage behavior.

Include the semantic stage mode and causal-stage selection in fingerprints and manifests so that:


and:

```text
STAGES=dataset-extraction
```

cannot collide, resume each other, or reuse each other’s completion records merely because their top-level router stage lists are identical.

When the router launches dataset child processes for `--dataset both`, propagate the original semantic mode explicitly. Do not rely on serializing only the expanded stage list.

## Causal `main.py` design

Add a CLI option such as:

```text
--causal-stages
```

Supported forms must include:

```text
all
graph
graph,majority_vote
```

Implement this as an ordered-prefix selector.

The valid causal sequence is:

```python
[
    "graph",
    "majority_vote",
    "mortality_prediction",
    "matching",
    "cate_estimation",
    "analyze_cate_results",
    "permutations_test",
]
```

A selected causal scope must be a non-empty prefix of this sequence.

Accept:

```text
graph
graph,majority_vote
graph,majority_vote,mortality_prediction
...
all
```

Reject non-prefix, reordered, duplicated, or unknown selections, including:

```text
majority_vote
graph,matching
majority_vote,graph
graph,graph
```

Use clear errors that show the valid stage order.

Default behavior must be `all`, preserving backward compatibility for direct callers that do not pass the new option.

Store both the selected causal stages and the original selector in the run context and summary.

2. verify the graph pickle and graph PNG;
